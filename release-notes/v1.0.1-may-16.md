# Figlink v1.0.1 вҖ” Visual Agent

**May 16, 2026**

Screenshots arrive. The AI can now *see* your Figma canvas вҖ” not just read JSON. Eleven new screenshot tools plus viewport awareness let the AI visually inspect selections, search the canvas autonomously, screenshot specific nodes by link, and scroll the viewport to share context with you.

Also: spatial coordinates are now included in every node serialization, the MCP tool count was consolidated from 85 to 76, and the auto-start script now launches ngrok alongside the server.

---

## New Feature: Visual Agent (Screenshots)

The core addition. Screenshots are an *additional data modality* alongside existing text-based tools вҖ” all previous tools continue working identically. The AI receives real rendered images directly through the MCP protocol.

### Design Principles

- **Additive, not replacing.** All 70+ text tools unchanged. Screenshots are extra.
- **Deduplication via ancestor walking.** Multi-image tools never export redundant screenshots. If you select a Frame and a Button inside it, you get one image (the Frame) вҖ” the Button is already visible.
- **Pagination, not caps.** Gallery tools use `page`/`pageSize`/`hasMore` so the AI can request more without artificial limits.
- **Concurrent exports.** Multi-image tools batch exports in groups of 5 for speed while respecting Figma's API.
- **Partial-failure handling.** If one node in a gallery fails to export, the others still come through вҖ” failed items become text error messages instead of crashing the entire response.

### Screenshot Tools (11 new)

| Tool | Description |
|------|-------------|
| `figma_screenshot_selection` | Screenshot everything currently selected. Multi-selection exports each node individually with deduplication and pagination. Spatial summary included. |
| `figma_screenshot_node` | Screenshot a specific node by ID. Works on any type: frames, components, vectors, text, etc. |
| `figma_screenshot_by_link` | Paste a Figma URL вҶ’ AI screenshots the node it points to. Parses the URL, finds the node, captures it in one call. |
| `figma_screenshot_page_overview` | Birds-eye view of the entire current page. Falls back to exporting top-level frames as a gallery if full-page export fails. |
| `figma_find_and_screenshot` | Search for nodes by name/type/text content and screenshot all matches. AI can autonomously discover and visually inspect elements. Paginated with yield points for large files. |
| `figma_screenshot_frame_thumbnails` | Thumbnail gallery of every top-level frame on the page. AI surveys the file at a glance. Paginated. |
| `figma_screenshot_node_with_context` | Screenshot a node PLUS its containing frame. Shows both the detail and its layout surroundings in one response. |
| `figma_screenshot_viewport_region` | Screenshot what the user currently sees on their Figma canvas. Captures all frames overlapping the viewport. Deduplicated. Dominant-frame optimization: if one frame fills >80% of the viewport, returns just that one. |

### Viewport & Canvas Management (3 new)

| Tool | Description |
|------|-------------|
| `figma_get_viewport_info` | Current viewport center, zoom level, and visible bounds. AI knows what area of the canvas you're looking at. |
| `figma_find_visible_nodes` | List which top-level frames are currently visible in the viewport. Returns IDs, names, and bounding boxes. |
| `figma_scroll_to_node` | Scroll the Figma canvas to center on a specific node. Creates shared visual context вҖ” the AI can show you what it's looking at. |

## Improvement: Spatial Coordinates in Node Data

All nodes returned by `figma_get_selection`, `figma_get_nodes`, and `figma_get_nodes_flat` now include `x`, `y`, `width`, and `height`. This was the highest-impact existing gap вҖ” the AI was spatially blind from text data alone.

```
// Before: { id: "1:5", name: "Button", type: "FRAME" }
// After:  { id: "1:5", name: "Button", type: "FRAME", x: 200, y: 100, width: 160, height: 48 }
```

This benefits both text-based and screenshot tools. Spatial summaries in screenshot galleries are now meaningful ("Header at (0,0), Sidebar at (0,800), Footer at (0,5000)").

## Improvement: Tool Consolidation (85 вҶ’ 76)

Single-use and bulk variants merged into unified tools. The `_by_key` pattern folded into parent tools via optional `styleKey`/`variableKey` fields.

| Before (removed) | Kept | What changed |
|---|---|---|
| `figma_bulk_rename` | `figma_rename_node` | Accepts `items[]` array вҖ” 1 item or many |
| `figma_bulk_set_characters` | `figma_set_characters` | Accepts `items[]` array |
| `figma_bulk_apply_text_style` + `_by_key` | `figma_apply_text_style` | Each item uses `styleId` OR `styleKey` |
| `figma_bulk_apply_fill_variable` + `_by_key` | `figma_apply_fill_variable` | Each item uses `variableId` OR `variableKey` |
| `figma_bulk_set_variable_binding` + `_by_key` | `figma_set_variable_binding` | Each item uses `variableId` OR `variableKey` |
| `figma_bulk_set_property` | `figma_set_property` | Accepts `items[]` array |
| `figma_set_style_property` (singular) | upgraded to `items[]` | No tool removed, just made plural |
| `figma_set_style_variable_binding` (singular) | upgraded to `items[]` | No tool removed, just made plural |
| `figma_delete_style` (singular) | upgraded to `styleIds[]` | No tool removed, just made plural |
| `figma_duplicate_text_style` (singular) | upgraded to `items[]` | No tool removed, just made plural |

The AI no longer needs to choose between "am I doing one thing or many?" вҖ” just pass an array.

## Improvement: Generalized Instance Swapping

`figma_swap_button_instances` was too narrow вҖ” it could only find instances with "button" in their name. It's been replaced by a generalized tool:

| Tool | Description |
|------|-------------|
| `figma_swap_instances` | Find all instances whose name matches a `searchPattern` (e.g. `"icon"`, `"card"`, `""` for all) and swap them to a new component set. Preserves text content, matches variant properties by name parsing. |

`figma_swap_button_instances` has been removed. Use `figma_swap_instances` with `searchPattern: "button"` (the default) for the same behavior.

## Improvement: Export & Base64 Performance

- **Chunked base64 encoding** (`uint8ToBase64`) replaces the old character-by-character loop. Uses 32KB chunks via `String.fromCharCode.apply()` вҖ” avoids O(nВІ) garbage collection on large exports.
- **Auto-scale-down:** If a node's longest dimension exceeds 4000px at the requested scale, the scale is automatically reduced to keep exports manageable. The actual scale used is reported in metadata.
- **`exportNode` refactored** to delegate to `exportNodeAsImage` вҖ” single source of truth for all export operations. Backward compatible; `figma_export_node` still returns the same format.

---

## Improvement: Auto-Start ngrok

`start-mcp.bat` now opens **3 terminals** instead of 2:

| Terminal | What runs |
|----------|-----------|
| Figlink Server | WebSocket relay on ws://localhost:9001 |
| ngrok Figlink | `ngrok http 39399` (auto-started if ngrok is in PATH) |
| Figlink MCP | MCP server on http://localhost:39399 |

If ngrok isn't installed, the script prints a notice instead of crashing. The MCP port was changed from 3000 to **39399** to avoid collisions with other local tools.

## Quick Start (updated)

### Daily use

**Native IDE:** Double-click `Windows Start Figlink.bat` or `Mac Start Figlink.command` in the `figlink-codebase` folder, then run the plugin in Figma.

**MCP вҖ” Local IDE:** Double-click `start-mcp.bat` or `start-mcp.command` in the `figma-mcp` folder, then add `http://localhost:39399/` to your IDE's MCP configuration.

**MCP вҖ” Web AI:** Same as local MCP. The start script auto-launches `ngrok http 39399`. Copy the forwarding URL from the ngrok terminal into your web AI's MCP settings.

# Figlink v1.0.0 — Initial Release

**May 16, 2026**

The first public release of Figlink. Let your AI assistant read and write Figma files — no API token required, no browser automation, no cloud dependency. Everything runs on your machine.

---

## What is Figlink?

Figlink is a bridge between AI assistants and Figma Desktop. It works through a lightweight Figma plugin that communicates over a local WebSocket relay to either a terminal-based CLI or a fully-featured MCP (Model Context Protocol) server.

Your AI gets direct access to your Figma canvas: read node trees, search for components, bind variables, rename layers, apply styles, create design tokens, and more — all through structured, typed tools that the AI discovers and calls autonomously.

---

## Three ways to connect

| Mode | How | Best for |
|------|-----|----------|
| **Native IDE** | AI runs commands directly from your IDE's terminal via WebSocket | Cursor, Windsurf, Trae, VS Code with terminal access |
| **MCP — Local IDE** | AI connects to `http://localhost:3000` via MCP | Claude Desktop, Cursor, VS Code Copilot, any MCP client |
| **MCP — Web AI** | Same server exposed via ngrok to the internet | Notion AI, Claude web, ChatGPT, any web-based AI platform |

All three modes use the same plugin and link server. Install the plugin once and choose the mode that fits your workflow.

## Architecture

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌───────────┐
│  AI Assistant │ ──▶ │  MCP Server  │ ──▶ │  Link Server │ ──▶ │  Plugin   │
│  (IDE / Web)  │ ◀── │  :3000 HTTP  │ ◀── │  :9001 WS    │ ◀── │  in Figma │
└──────────────┘     └──────────────┘     └──────────────┘     └───────────┘
```

- **MCP Server** (`figma-mcp/server.js`) — HTTP server exposing Figma operations as 74 typed MCP tools with Zod-validated parameters. Multi-session: each AI client gets its own transport and session.
- **Link Server** (`figlink-codebase/link-server/server.js`) — WebSocket relay that routes commands from clients to the correct Figma plugin instance. Supports up to 8 concurrent files.
- **Plugin** (`figlink-codebase/figma-plugin/code.js`) — Runs inside Figma Desktop. Receives commands, executes them against the Figma Plugin API, and returns results.
- **Bridge** (`figma-mcp/bridge.js`) — WebSocket client that connects the MCP server to the link server. Handles reconnection and request/response matching.

## Key Features

### Multi-Session MCP Architecture

The MCP server creates a new transport and McpServer instance per client session. This means:
- Multiple AI clients can connect simultaneously
- Each client gets its own session with independent state
- Sessions are cleaned up automatically when closed
- Re-initialization creates a fresh session (no "server already initialized" errors)

### No Figma API Token Required

Figlink uses the Figma Plugin API, which runs inside the Figma Desktop app. There is no REST API, no access tokens, no OAuth — just install the plugin and connect.

### Web-Compatible (ngrok)

The MCP server handles OAuth/OIDC discovery gracefully:
- `/.well-known/openid-configuration` → returns 404 (tells the client "no OAuth here")
- `/.well-known/oauth-authorization-server` → returns 404
- `/oauth/*` → returns 404 + CORS preflight
- Any URL path (e.g., `/model_context_protocol/2025-03-26/mcp`) is accepted identically to `/`

### Generous Timeouts for Large Files

- **Bridge timeout:** 30 minutes (was 2 minutes)
- **Link server TTL:** 30 minutes (was 10 minutes)
- **Plugin walks:** Asynchronous with periodic event-loop yielding every 1000 nodes — prevents Figma from killing the plugin for blocking the UI thread
- **Bulk operation tools:** Have their own long timeouts for large batch jobs

## Quick Start

### One-time setup

```bash
# Clone the repo
git clone https://github.com/danielfransix/figlink.git

# Install dependencies
cd figlink-codebase/link-server && npm install
cd ../../figma-mcp && npm install

# Install the Figma plugin
# Figma → Plugins → Development → Import plugin from manifest
# → Select figlink-codebase/figma-plugin/manifest.json
```

### Daily use

**Native IDE:** Double-click `Windows Start Figlink.bat` or `Mac Start Figlink.command` in the `figlink-codebase` folder, then run the plugin in Figma.

**MCP — Local IDE:** Double-click `start-mcp.bat` or `start-mcp.command` in the `figma-mcp` folder, then add `http://localhost:3000/` to your IDE's MCP configuration.

**MCP — Web AI:** Same as local MCP, plus `ngrok http 3000` in a terminal. Paste the forwarding URL into your web AI's MCP settings.

## Known Limitations

- **Library component enumeration** requires the Figma REST API. `figma_get_library_components` returns guidance on using `figma_execute` with `importComponentByKeyAsync` instead.
- **Comments** (get/post/delete) are not available through the Plugin API — they require the REST API.
- **Version history** operations require the REST API.
- **Accessibility audits** require a browser-based approach (not the Plugin API).
- **ngrok free tier URLs** change on every restart. Upgrade to a paid plan for a fixed reserved domain.
